Это не просто ещё одно приложение с заметками на карте.  
«Места» — это персональная ГИС со структурой, предназначенной для долгосрочного и системного управления географическими данными.

## Основная идея

В основе всего лежат **атомарные «Точки»**.  
Точки переиспользуются и могут служить базой для более сложных сущностей, таких как:

- Места
- Маршруты
- другие пространственные структуры

Это формирует **нормализованную пространственную модель** вместо разрозненных и дублирующихся данных.

## Почему это важно

Большинство приложений работает по принципу: `маркер → текст`.

«Места» рассматривают данные как систему:

- переиспользуемые координаты
- связанная структура
- согласованные пространственные ссылки

Это делает данные масштабируемыми, комбинируемыми и устойчивыми к изменениям в будущем.

![UI: Distance measurement mode](public/raw/places-screenshot-1.png)

## Модель синхронизации

«Места» используют **детерминированную пакетную систему синхронизации**:

- сущности отслеживаются как `added / updated / deleted`  
- изменения передаются в виде структурированных пакетов
- сервер обрабатывает их предсказуемым образом

Такой подход:

- упрощает управление состоянием
- исключает скрытые изменения
- подготавливает систему к **offline-first сценариям**

## Технологии

- Frontend: Vue.js + Pinia
- Backend: PHP + MariaDB / MySQL
- Карты: OpenStreetMap, Yandex.Maps

## Дорожная карта

- Удобный сбор данных в реальном мире с мобильных устройств (GPS, фото, быстрое создание мест)
- PostgreSQL + PostGIS (расширенные пространственные запросы)
- улучшенная поддержка офлайн-режима
- импорт и экспорт данных
- API и интеграции

## Философия

«Места» создаются как **альтернатива проприетарным картографическим платформам с приоритетом приватности**.

Ваши данные принадлежат вам.  
Вы контролируете свою инфраструктуру.

## Статус

Проект находится в стадии beta, но уже полностью работоспособен и используется на практике.

## Установка и развертывание

1.  **Клонирование:** `git clone https://github.com/assador/places.git`
2.  **База данных:** Создайте БД и импортируйте дамп из `/mezzanine/db_places.sql`.
3.  **Настройка:** Отредактируйте параметры в `/src/shared/constants.js` и `/backend/config.php`.
4.  **Права доступа:** Убедитесь, что директории в `/dist/uploads/images/` доступны для записи.
5.  **Cron:** Настройте запуск `/backend/dist/cron.php` по расписанию для очистки временных данных.
6.  **Сборка:**
    ```bash
    npm install
    npm run build
    ```
7.  **Сервер:** Настройте веб-сервер, указав `/dist` в качестве корневой директории.
